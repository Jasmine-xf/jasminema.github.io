UofT Library Navigation

Navigating the University of Toronto’s St. George campus is often challenging due to complex building layouts, inconsistent signage, and limited accessibility support. Existing tools such as Google Maps or static floor plans cannot provide accurate, room-level guidance, leaving many students, visitors, and individuals with accessibility needs struggling to find their way efficiently.

This project investigates the barriers to campus wayfinding and defines the high-priority requirements for an indoor navigation system tailored to UofT. Based on surveys, interviews, and thematic analysis, we identified six core needs — real-time positioning, accessible routes, room-level search, clear landmarks, multilingual support, and an intuitive campus map — leading to a proposed solution that reduces navigation time, improves accessibility, and enhances the overall campus experience.

Desk Research

To understand how people navigate complex indoor spaces and what challenges they face, our team conducted desk research as part of collaborative project with Mobile Application Development Lab (MADLab) at the University of Toronto

The goal was to determine what practices work well, what challenges remain unsolved, and how these insights could inform the design direction for UofT's indoor navigation system.

How We gonna do that?

  • Reviewed 10+ reports and navigation tools to examine common usability and accessibility issues.
  • Compared how different systems approach real-time positioning, visual cues, and accessibility features.
  • Mapped out key opportunities that informed our primary research plan and guided later design decisions.

Key Findings

  • 3D/AR navigation and real-time tracking are industry standards, but few solutions integrate accessibility features such as screen readers, high-contrast modes, or barrier-free routes.
    → There's a gap in inclusive indoor navigation design.
  • Information about indoor spaces is fragmented and static. Most tools rely on outdated floor maps or scattered signage, which causes confusion and inconsistency across buildings.
    → There's a need for a centralized, real-time navigation system that updates continuously.
  • User confidence and independence are limited. Research shows that many people—especially newcomers and those with accessibility needs—depend on asking others for directions.
    → There's an opportunity to build an experience that supports independent navigation through clearer visual and contextual cues.
  • Offline and multi-language functions are underdeveloped. Few systems support use without internet access or offer multilingual interfaces.
    → There's potential for offline usability and language inclusivity in UofT's campus environment.

Primary Research

To make the UofT Library Navigation system not only functional but also intuitive and accessible, I established a set of design decisions and content guidelines. These were informed by both primary research (user interviews & usability testing) and secondary research (best practices & accessibility standards).

Visual landmark-based navigation: Use clear visual landmarks and directional cues instead of complex text-based directions to help users navigate intuitively.

Step-by-step guidance: Break down complex navigation into manageable steps with clear progress indicators and contextual information at each stage.

Real-time data integration: Connect with library inventory systems to provide accurate availability and location information, preventing wasted trips.

Multi-stop route optimization: For users finding multiple resources, optimize routes to minimize walking time and provide efficient path planning.

Accessibility considerations: Design for users with different mobility needs, including clear pathways and alternative route options.

Universal interface consistency: Maintain consistent design patterns across different library buildings to reduce learning curve and improve user confidence.

Result Analysis

Indoor Positioning System

For this project, I implemented an indoor positioning system using React Native with location services integration. The system provides accurate positioning within library buildings and enables precise navigation guidance.

  1. Developed a comprehensive database of library layouts, call number mappings, and landmark locations.
  2. Implemented a routing algorithm that considers floor changes, accessibility requirements, and real-time space availability.
  3. Built a prototype data pipeline using Python for processing library inventory and location data.

User Interface Design

I designed a mobile-first interface that prioritizes clarity and ease of use. The interface provides clear visual hierarchy, intuitive navigation controls, and contextual information. The design accommodates different user types and navigation scenarios while maintaining consistency across library buildings.

Usability Testing

The UofT Library Navigation project taught me that designing for indoor wayfinding requires a deep understanding of spatial relationships and user behavior in physical environments. Working with complex building layouts showed how important it is to provide clear visual guidance and break down navigation into manageable steps. I also learned how critical real-time data integration is for building user trust and ensuring system accuracy. Most importantly, this project reinforced the value of user research in understanding diverse navigation needs and the importance of designing for different user types and scenarios.
